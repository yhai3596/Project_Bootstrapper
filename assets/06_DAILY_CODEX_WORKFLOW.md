# Alan 的 Codex 日常开发流程

## 一、每天开始前：项目状态检查

```text
请先不要修改代码。请检查当前项目状态：
1. 当前分支和未提交修改
2. 项目结构
3. 最近可能相关的文件
4. 如何运行
5. 如何测试
6. 今天最适合先做的 1-3 个任务
```

## 二、开发单个功能：限定边界

```text
本次只做以下功能，不做额外扩展：

【目标】
<填写>

【输入】
<填写>

【输出】
<填写>

【验收】
<填写>

请先判断是否过度工程，然后给最小实现方案。确认方案后再改代码。
```

## 三、每次改完：强制验证

```text
请运行当前可行的测试/构建/最小验证命令。
如果无法运行，请说明原因，并给我一条我可以在 Windows PowerShell 里复制执行的验证命令。
```

## 四、每次收尾：交付总结

```text
请按以下格式总结本轮开发：
1. 修改文件
2. 修改目的
3. 运行命令
4. 验证结果
5. 风险
6. 下一步建议
```

## 五、推荐节奏

```text
项目体检 → 任务拆解 → 单任务执行 → 验证 → 复盘 → 下一个任务
```

不建议：

```text
直接说“帮我把整个项目做完”
```

建议：

```text
现在只执行 task_queue.md 中第 1 个任务。
请先说明目标、计划修改文件、验证命令，等我确认后再动手。
```
