# Codex 常用任务提示词

## 1. 项目体检

```text
请先不要修改代码。请阅读当前项目结构，输出：
1. 项目目标判断
2. 技术栈判断
3. 关键入口文件
4. 如何运行
5. 潜在风险
6. 最小可验证路径
7. 你建议下一步先做什么
```

## 2. 新功能开发

```text
请按最小可行版本实现以下功能：

【功能目标】
<写清楚用户、输入、输出、验收标准>

【约束】
- 不要大规模重构
- 不要引入新依赖，除非必要
- Windows PowerShell 可运行
- 修改后运行可行的测试或最小验证命令

请先给实施计划，再开始修改。
```

## 3. Bug 修复

```text
请定位并修复以下问题：

【现象】
<粘贴完整报错>

【复现步骤】
<如何触发>

【期望结果】
<应该发生什么>

要求：
1. 先解释可能根因
2. 再定位相关文件
3. 最小修改修复
4. 给出验证命令
5. 说明回归风险
```

## 4. 代码审查

```text
请作为严格 Code Reviewer 审查当前未提交修改，重点检查：
1. 是否解决真实问题
2. 是否过度工程
3. 是否引入不必要依赖
4. 是否有安全风险
5. 是否有 Windows 兼容问题
6. 是否有数据破坏风险
7. 测试是否充分
8. 文档是否同步

输出：
- 必须修改的问题
- 建议修改的问题
- 可以暂缓的问题
- 最终是否建议合并
```

## 5. 长任务拆解

```text
请把这个开发任务拆成可连续执行的任务队列：

【目标】
<填写目标>

要求：
1. 每个任务 30-90 分钟内可完成
2. 每个任务有输入、输出、验收标准
3. 先做能验证价值的主路径
4. 不要先做复杂架构
5. 输出 Markdown 任务列表
```

## 6. 收尾总结

```text
请按以下格式总结本轮开发：
1. 修改文件
2. 修改目的
3. 运行命令
4. 验证结果
5. 风险
6. 下一步建议
```
